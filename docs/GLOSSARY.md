# Глоссарий

Термины, которые в документации чаще всего остаются на английском. Рядом дана русская формулировка.

| Термин | Значение |
|--------|----------|
| **P2P** (peer-to-peer) | Связь напрямую между узлами без центрального сервера приложений. Rendezvous используется только для первичного знакомства адресов. |
| **Узел** (node) | Запущенный экземпляр программы с UDP-сокетом и логикой протокола. |
| **Handshake** | Обмен ключами перед шифрованным трафиком. У нас: Noise XX, три сообщения. |
| **Noise** | Библиотека и семейство протоколов шифрования. Паттерн XX не требует заранее известных статических ключей. |
| **Session** | Сессия шифрования после handshake: пара ключей для отправки и приёма. |
| **Frame** | Единый UDP-пакет протокола Nyx: заголовок + полезная нагрузка. |
| **Stream** (поток) | Логический канал внутри одного соединения. Поток 0 зарезервирован под служебные сообщения. |
| **Multiplexer** (мультиплексор) | Разделение одного шифрованного канала на несколько потоков. |
| **Rendezvous** | Вспомогательный UDP-сервер: хранит временную привязку «токен приглашения → адрес узла». |
| **Invite token** | Случайные 32 байта. Узел-слушатель публикует их на rendezvous; подключающийся узел ищет по ним адрес. |
| **Endpoint hint** | Подсказка адреса: IP, порт и случайный nonce. Не привязана к личности пользователя. |
| **Hole punch** | Серия UDP-запросов, чтобы «пробить» NAT и открыть путь между узлами. |
| **ARQ** (Automatic Repeat reQuest) | Подтверждения и повторная отправка потерянных пакетов. Реализован selective repeat. |
| **MTU** | Максимальный размер полезной нагрузки одного фрагмента (сейчас 1200 байт). |
| **Ping / Pong** | Проверка живости: ping отправляет nonce, pong возвращает тот же nonce. |
| **MVP** | Минимально жизнеспособный продукт: два компьютера обмениваются сообщениями и файлами через P2P. |
| **Discovery** | Поиск других узлов или пользователей в сети без центрального каталога контактов. |
| **Nickname** | Читаемый псевдоним пользователя. Не равен криптографическому ключу. |
| **Поле** (field, group) | Групповой чат MVP: несколько участников, общая история, общие политики файлов. Создаётся пользователем; новых участников добавляют по **group invite**. Синоним в коде: `Group`, `GroupId`. |
| **GroupId** | Уникальный идентификатор поля (32 байта). Не равен invite token. |
| **Group invite** | Токен для входа в поле (аналог invite token для listen, но привязан к группе). |
| **ChatId** | Идентификатор переписки: личка (два UserId) или поле (GroupId). Различает историю и маршрут сообщений. |
| **Личка** (DM) | Чат один-на-один между двумя узлами. Сейчас реализовано (фазы 1–3). |
| **Индексация** | Сканирование выбранных папок и построение каталога файлов для быстрой выдачи в сеть. |
| **FilePermission** | Битовая маска прав на файлы в поле: список, скачивание, отправка, открытие по сети, управление папками и ролями. |
| **FileRole** | Именованный набор `FilePermission` для участников поля; хранится в `file_access.json`. |
| **QML** | Язык разметки Qt Quick для declarative UI. |
| **AppCore** | C++ слой между QML и libnyx: модели чатов, отправка сообщений, статусы. |
| **Design tokens** | Именованные константы цветов, шрифтов, отступов для единого стиля. |
