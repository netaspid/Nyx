# Руководство администратора Nyx

Nyx — P2P-мессенджер: лички, поля (группы), обмен файлами. Сообщения и файлы идут напрямую между узлами. Сервер rendezvous нужен только чтобы два узла за NAT нашли адреса друг друга.

## Состав дистрибутива

| Компонент | Назначение |
|-----------|------------|
| `nyx-app` / `NyxSetup.exe` | Клиент для пользователей |
| `nyx-rendezvous` | Bootstrap UDP (по умолчанию порт 3478) |
| `nyx-node` | CLI для диагностики |

Пользователям обычно достаточно установщика `NyxSetup.exe`.

## Установка клиента (Windows)

1. Собрать проект (см. корневой README) или взять готовый `build\NyxSetup.exe`.
2. Запустить установщик на ПК пользователя.
3. При первом запуске создать аккаунт и сохранить recovery-фразу.
4. В **Настройки → Сеть** указать rendezvous: `host:3478` (несколько через запятую).
5. Режим обнаружения: **Авто** (LAN + интернет) или **Только интернет**.

Данные аккаунта: `%APPDATA%\Nyx\` (профили, ключи, история, индекс файлов).

## Развёртывание rendezvous

Протокол: **UDP**. Порт по умолчанию: **3478**.

Клиенты не гоняют через него чат — только register/lookup invite token.

### Linux (systemd)

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target nyx-rendezvous

sudo ufw allow 3478/udp
sudo useradd -r -s /usr/sbin/nologin nyx
sudo install -m755 build/nyx-rendezvous /opt/nyx/
```

`/etc/systemd/system/nyx-rendezvous.service`:

```ini
[Unit]
Description=Nyx rendezvous
After=network-online.target

[Service]
Type=simple
ExecStart=/opt/nyx/nyx-rendezvous --bind=0.0.0.0:3478 --rate-limit=240
Restart=always
RestartSec=3
User=nyx

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now nyx-rendezvous
```

### Windows

```powershell
.\build\nyx-rendezvous.exe --bind=0.0.0.0:3478 --rate-limit=240
```

Открыть входящий UDP 3478 в брандмауэре.

### Docker

Образ и пример запуска — в каталоге `deploy/rendezvous/` репозитория (если используется).

## Настройка сети у клиентов

1. Все клиенты одной организации должны указывать **один и тот же** список rendezvous.
2. Кнопка **Проверить** в настройках сети должна успешно достучаться до сервера.
3. В LAN два клиента могут соединиться без rendezvous (локальное обнаружение).

## Поля и файлы

- Поле создаёт владелец; участники входят по invite.
- Пока hub владельца не в сети, участники не пишут в поле и не видят живой каталог файлов.
- Общие папки поля смотрят во вкладке **Файлы → Ресурсы** (не в «Обзор» — там только свои папки).
- Права на файлы задаёт владелец на вкладке **Доступ** (роли Участник / Только чтение и свои).
- После удаления или повторного добавления папки у владельца участнику нужно нажать **Обновить** в шапке «Файлы» (список сбрасывается к корням поля).
- Крупные папки индексируются в фоне: в экране «Файлы» виден прогресс сканирования.
- В «Ресурсы» кнопка «вверх» / крошка **Ресурсы** возвращает к списку share-папок поля.
- Кнопка **Передать** в «Обзор» есть только для личных файлов в 1:1 чате (отправить файл собеседнику). В поле общий каталог смотрят во вкладке «Ресурсы».
- Очень большие каталоги (тысячи имён в одном уровне, например `node_modules`) отдаются урезанным списком: сначала папки, чтобы UI и канал не падали на лимите шифрования.

## Диагностика

| Симптом | Что проверить |
|---------|----------------|
| Lookup failed | Доступность UDP 3478, один и тот же host:port у обеих сторон, firewall |
| Поле «не в сети» у участника | Запущен ли hub у владельца, включён ли автозапуск сессий |
| Пустые «Ресурсы» | Клиент в поле (Live), область = нужное поле, кнопка «Обновить»; если только что удалили папку у владельца — «Обновить» ещё раз |
| Краш / пусто после удаления папки | У участника: «Обновить» (сброс browse). У владельца: перезапуск не нужен, индекс локальный |
| Не ставится клиент | Запуск `nyx-app.exe` из консоли, смотреть текст ошибки Qt |

CLI для проверки:

```powershell
.\build\nyx-node.exe --help
.\build\nyx-rendezvous.exe --bind=127.0.0.1:3478
```

## Резервное копирование

Критично для пользователя: recovery-фраза аккаунта.  
Для админа на ПК: каталог `%APPDATA%\Nyx\` (копия при переносе профиля).

На сервере rendezvous состояние эфемерное — достаточно перезапуска бинарника и открытого UDP.
